About Me

안녕하세요, 웹 개발자 서예진입니다. 저는 기술을 통해 복잡한 문제를 단순화하고, 사람들에게 실질적인 도움을 주는 서비스를 만드는 데 열정을 가지고 있습니다. 특히 의료 분야와 같이 전문성이 요구되지만 사용자의 접근성이 중요한 영역에서, 기술적 장벽을 낮추는 사용자 친화적인 인터페이스를 설계하는 것을 목표로 합니다.

프로젝트 Care Note는 이러한 저의 가치관이 담긴 결과물입니다. 환자에게는 편안한 소통의 창구를, 의료진에게는 업무 효율을 높여주는 도구를 제공함으로써 의료 서비스의 질적 향상을 도모했습니다.

앞으로도 끊임없이 새로운 기술을 학습하고 적용하며, 사용자의 피드백을 바탕으로 서비스를 개선해 나가는 성장형 개발자가 되겠습니다. 언제 어디서나 접근 가능한 웹 기술의 장점을 살려 더 넓은 세상과 소통하고 싶습니다.

Project Overview

Care Note (케어노트)

웹 기반 AI 산부인과 문진 시스템

의사와 환자를 잇는 AI 스마트 리포트 화면

문제 정의 및 해결

  • 문제점: 짧은 진료 시간, 반복적인 문진 과정, 환자의 말하기 어려운 고민 등 의료 현장의 비효율과 소통 부재.
  • 솔루션: 별도 앱 설치 없는 웹 기반 문진 시스템 구축. AI를 활용하여 환자의 증상을 사전 분석하고 요약 리포트를 제공.
  • 차별점: Node.js와 Gemini Pro를 연동한 실시간 분석, Firebase를 이용한 Serverless 실시간 데이터 동기화.

핵심 기능 (Core Features)

Care Note가 제공하는 3가지 주요 솔루션입니다.

AI 스마트 문진 리포트

Google Gemini Pro 모델을 활용하여 환자가 작성한 방대한 양의 텍스트를 의사가 한눈에 파악할 수 있도록 요약합니다.

  • 자동 요약 (Summary): 환자의 증상과 상태를 의학적 관점에서 요약 정리.
  • 핵심 키워드 (Keywords): 주요 증상을 태그 형태로 시각화.
  • 감성 분석: 환자의 심리적 상태 파악 보조.

증상 시각화 대시보드

텍스트만으로는 파악하기 힘든 통증의 강도나 증상의 분포를 Chart.js를 이용해 시각적인 그래프로 제공합니다.

  • 직관적 파악: 차트의 모양(찌그러진 정도 등)만 보고도 환자의 상태를 1초 만에 파악.
  • 데이터 정량화: 척도형 답변 점수를 데이터로 변환하여 실시간 렌더링.

디지털 하이브리드 워크플로우

Firebase Firestore를 도입하여 환자와 의사 간의 데이터를 실시간으로 동기화합니다. 종이 문진표의 번거로움을 디지털로 완벽하게 전환했습니다.

  • 실시간 상태 변경: 대기 중 -> 작성 중 -> 작성 완료 상태가 의사 대시보드에 즉시 반영.
  • 접근성: 별도의 앱 설치 없이 QR코드나 링크로 접속 가능.

기술 아키텍처 (Tech Stack)

안정적이고 확장 가능한 서비스를 위해 최신 웹 기술을 사용했습니다.

Care Note System Architecture Overview

Frontend

  • HTML5 & JavaScript: 시맨틱 마크업 및 동적 기능 구현
  • Tailwind CSS: Utility-first 기반의 빠르고 유연한 스타일링, 모바일/PC 반응형 완벽 지원
  • Chart.js: 데이터 시각화 및 증상 그래프 구현

Backend & AI

  • Node.js (Express): REST API 서버 구축 및 AI 요청 처리
  • Google Gemini Pro: 의료 텍스트 분석, 요약, 키워드 추출을 위한 생성형 AI 모델 활용

Database & Infra

  • Firebase Firestore: NoSQL 기반 실시간 데이터베이스, 서버 구축 비용 절감
  • Serverless: 유지보수가 용이하고 확장성 높은 클라우드 환경
